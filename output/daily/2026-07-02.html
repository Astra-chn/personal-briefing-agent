  4. GitHub 项目 fanbox：Vibe Coding 驾驶舱（文件+终端+实时改动）
    来源: GitHub | 链接
    为什么重要： 它把文件浏览、命令执行、代码改动过程整合到一个界面，让你“俯瞰”每一次修改——很适合边学边改的初学者。
    是否适合你学习： 非常适合。用 JavaScript 写的，Star 数高（839），社区活跃。你可以直接 fork 下来，一边看它如何监听文件变化、一边修改自己项目。它本身就是个“通过项目学习”的绝佳例子。

对我的个人建议

  1. 立刻上手 fanbox 项目
    - Fork 仓库，看 README 启动教程，体验“vibe coding”模式。
    - 学会后可以用这个工具来管理你自己的小项目（比如写一个 RAG 聊天机器人）。

  2. 用 toolport 的架构思路改进自己的工具链
    - 不必现在写 Rust，先在纸上画出你当前使用的 AI 工具（如 Ollama、LangChain、FireCrawl）如何通过 MCP 统一管理，理解“懒加载发现”和“工具隔离”的思想。

  3. 阅读 OpenAI 那篇 Agent 研究论文
    - 即使不读全文，至少看摘要和结论。把其中提到的“长任务分解”“多步推理”等概念记下来，下次设计自己的 Agent 时直接套用。

  4. 关注武汉光谷的招聘动态
    - 在智联、猎聘搜索“智能体开发”“Agent 工程师”，了解企业具体技能要求（比如需要会 LangGraph、MCP、RAG 等），作为你下阶段学习的目标。

  5. 注意半导体涨价对云成本的影响
    - 如果你打算租用 GPU 训练/推理，近期优先使用按需实例，不要签长合同；可以对比下华为云、阿里云的国产芯片优惠方案。
