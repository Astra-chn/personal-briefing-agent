## 今日最值得关注的 5 件事

1. **n8n 开源自动化工作流平台**（GitHub ⭐193k）  
   - 适合你：可视化搭建 + 自定义代码，自带 AI 能力，零基础也能快速做出一个自动化工具（比如定时抓取新闻、发邮件）。

   - 链接：https://github.com/langgenius/dify

   - 趋势：Ona 提供安全、持久的云端环境，让 AI 代理能长时间执行复杂任务（比如自动处理审批、监控系统）。这意味着后端开发者的工作会越来越多被 AI 代理替代或辅助，但也催生了“Agent 运维”新岗位。  
   - 链接：https://openai.com/index/openai-to-acquire-ona

4. **System Prompts 合集**（GitHub ⭐140k）

   - 链接：https://github.com/x1xhlol/system-prompts-and-models-of-ai-tools

5. **JavaGuide：后端面试通识**（GitHub ⭐156k）

## AI 发展动态

- **OpenAI 推出“部署模拟”技术**：在发布模型前，用真实对话数据模拟部署场景，提前发现安全性问题。  
  → 趋势影响：AI 模型上线风险降低，企业更敢放心用 AI 客服、AI 代理做关键业务。对个人来说，未来会看到更多“AI 即服务”产品，后端开发会被抽象为 Agent 编排。
- **OpenAI 收购 Ona**：为 Codex 增加持久化云端环境，AI 代理可以连续工作几小时甚至几天（比如自动写代码、管理项目）。  
  → 直接影响：后端 API 设计、任务调度等行业可能被 AI 代理重构，但同时也需要懂 Agent 开发的新角色。

## 对我的个人建议

1. **本周动手项目**：用 Dify 搭建一个“RAG 知识库问答”工具。  
   - 步骤：注册 Dify（或本地部署）→ 上传你关注的公考/政策文档 → 创建一个 AI 助手 → 测试提问。整个过程只需拖拽，1 小时完成。  
   - 目标：理解 RAG 的核心（检索+生成），为后续深入学习打基础。

2. **每天 15 分钟**：刷 JavaGuide 中“RAG”“Agent”“分布式”三章的基础概念，不求背，只求知道“这是什么、用来干什么”。

3. **关注 n8n 自动化**：试试用 n8n 写一个“每天早上抓取 GitHub trending 并推送到微信”的自动化工作流。这能让你直观感受到自动化工具的威力，也符合你“后端转 AI”的路径——未来很多后端工作就是编排 Agent。

4. **公考准备建议**：在 Dify 的 RAG 知识库里放一份《2025 政府工作报告》+《科技政策汇编》，训练自己的 AI 助手帮你写申论素材或政策分析框架，事半功倍。

5. **长期方向**：保持对 OpenAI 收购 Ona 这类新闻的敏感——它预示了 AI Agent 会像“微服务”一样成为新一代基础设施。你现在的优势是：零基础反而没有旧技术包袱，可以直接从 Agent 和自动化工具入手，弯道超车。
